# Tie Hard — спецификация игры (MVP)

## Что за игра
Telegram Mini App: обучающая игра про завязывание узлов. Пользователь учит узел по пошаговой SVG-схеме, потом проходит квиз-проверку. Прогресс и достижения хранятся локально.

Стек: Vanilla JS, Vite, хостинг Vercel, без сервера.

## Состав MVP — 5 узлов
1. Бабочка
2. Грейпвайн
3. Ведёрный (Bucket Knot)
4. Стремя
5. Булинь

(Инструкции завязывания и тексты квизов сюда НЕ входят — это пишет контентщик позже.)

## Экраны
1. **Главный** — список из 5 узлов, у каждого статус: не начат / изучен / пройден квиз. Видно общий прогресс.
2. **Изучение узла** — пошаговая SVG-схема, листание шагов вперёд/назад, краткий текст шага.
3. **Квиз** — проверка по узлу после изучения (вопросы готовит гейм-дизайнер/контентщик).
4. **Результат квиза** — сдал/не сдал, отметка узла как пройденного, начисление достижения.
5. **Достижения** — список того, что открыто.

## Поток пользователя
Открыл бота → главный экран со списком → выбрал узел → пролистал схему по шагам → запустил квиз → сдал → узел отмечен пройденным, достижение начислено → вернулся на главный → следующий узел.

## Что хранится (Telegram CloudStorage, локально)
- Статус каждого узла: не начат / изучен / пройден
- Открытые достижения
- Общий прогресс (сколько узлов пройдено из 5)

## Достижения (MVP, базовые)
- Первый узел пройден
- Половина пути (3 из 5)
- Все 5 узлов пройдены

(Точный список и формулировки — за гейм-дизайнером.)
